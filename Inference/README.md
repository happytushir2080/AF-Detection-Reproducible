# ECG Inference Pipeline for Paper “XXX”

This repository contains the inference code used for the paper **“XXX”**.
The pipeline takes ECG records in CSV format, prepares patient-wise data, runs inference for a single patient/record, converts results into EDF-compatible formats, and generates combined statistics for all patients.

## Overview

The repository supports the following workflow:

1. Convert ECG records into CSV format.
2. Generate a `.txt` file containing paths to all available CSV records.
3. Extract patient-wise data from the CSV records.
4. Run inference using the trained model.
5. Convert ECG signals and annotations into EDF/EDF annotation format for visualization in **EDF Browser**.
6. Generate combined result statistics across patients.

## File Description

| File                              | Description                                                                                                                                                                          |
| --------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `generate_txt.py`                 | Takes a root directory containing ECG records in CSV format and generates a text file containing paths to the CSV files. It also creates a log file for missing or unreadable files. |
| `single_patient_data_split.py`    | Extracts/prepares patient-wise data from the CSV files listed in the `.txt` file generated by `generate_txt.py`.                                                                     |
| `data_extract.sh`                 | Shell script to run `single_patient_data_split.py` using the generated text file.                                                                                                    |
| `inference.py`                    | Runs inference for a single patient/record.                                                                                                                                          |
| `inference.sh`                    | Shell script to run `inference.py`. Input directories and paths can be changed directly inside the script.                                                                           |
| `convert_full_record_into_edf.py` | Converts full ECG records into EDF format so that they can be viewed in EDF Browser.                                                                                                 |
| `convert_full_record_into_edf.sh` | Runs `convert_full_record_into_edf.py` for all ECG records inside the given root directory.                                                                                          |
| `convert_into_edf_annotations.py` | Converts inference-generated annotation files into EDF annotation format for visualization in EDF Browser.                                                                           |
| `convert_into_edf_annotations.sh` | Runs `convert_into_edf_annotations.py` for all annotation files generated by the inference script.                                                                                   |
| `results_stats.py`                | Generates combined statistics for all patients after inference.                                                                                                                      |

## Input Data Format

The input ECG records should be available in **CSV format**.

If your ECG records are in WFDB format, you can convert them into CSV using the following repository:

[wfdb_to_csv](https://github.com/barthwalsaurabh0/wfdb_to_csv)

After conversion, the root directory should contain ECG records in CSV format.

Example:


```text
root_dir/
├── patient_001/
│   └── holter/
│       ├── 1/
│       │   └── record.csv
│       ├── 2/
│       │   └── record.csv
│       └── 3/
│           └── record.csv
├── patient_002/
│   └── holter/
│       └── 1/
│           └── record.csv
└── patient_003/
    └── holter/
        ├── 1/
        │   └── record.csv
        └── 2/
            └── record.csv


Each patient folder may contain a `holter` directory with one or more recording folders such as `1`, `2`, or `3`. It is not necessary that all patients have all three recordings.

```

## Installation

Clone this repository:

```bash
git clone <your-repo-url>
cd <your-repo-name>
```

Create and activate a Python environment:

```bash
conda create -n ecg-inference python=3.9
conda activate ecg-inference
```

Install the required packages:

```bash
pip install -r requirements.txt
```

If `requirements.txt` is not available, install the required dependencies manually based on the imports used in the scripts.

---

## Step 1: Generate CSV File List

Use `generate_txt.py` to scan the root directory and generate a text file containing paths to all ECG CSV files.

```bash
python generate_txt.py \
  --root_dir "path-to-root-dir" \
  --out_txt "path-to-output-txt" \
  --log_txt "path-to-log-txt"
```

Example:

```bash
python generate_txt.py \
  --root_dir /mnt/project2/DST-SERB/AIIMS-data \
  --out_txt csv_list.txt \
  --log_txt missing_log.txt
```

This will create:

```text
csv_list.txt
missing_log.txt
```

`csv_list.txt` contains the paths of valid CSV records.
`missing_log.txt` stores the missing/unreadable file information.

---

## Step 2: Extract Patient-wise Data

After generating the CSV list file, run the data extraction script:

```bash
./data_extract.sh "path-to-txt-file"
```

Example:

```bash
./data_extract.sh csv_list.txt
```

This script uses `single_patient_data_split.py` to prepare the data required for inference.

---

## Step 3: Run Inference

The file `inference.py` performs inference for a single patient/record.

Run inference using:

```bash
./inference.sh
```

Before running the script, update the required input/output directories inside `inference.sh`, such as:

```bash
INPUT_DIR="path-to-input-dir"
OUTPUT_DIR="path-to-output-dir"
MODEL_PATH="path-to-model-weights"
```

The inference script generates prediction outputs and annotation files for each processed patient/record.

---

## Step 4: Convert Full ECG Records into EDF Format

To visualize full ECG records in EDF Browser, convert the ECG records into EDF format.

```bash
./convert_full_record_into_edf.sh
```

This script runs `convert_full_record_into_edf.py` for all ECG records present in the root directory.

---

## Step 5: Convert Inference Annotations into EDF Annotation Format

After running inference, convert the generated annotation files into EDF annotation format.

```bash
./convert_into_edf_annotations.sh
```

This script runs `convert_into_edf_annotations.py` for all annotation files generated by the inference pipeline.

The generated EDF annotation files can be loaded into **EDF Browser** along with the EDF signal files.

---

## Step 6: Generate Combined Result Statistics

To generate combined statistics for all patients, run:

```bash
python results_stats.py
```

This script summarizes the inference results across all processed patients.

---

## Complete Pipeline

A typical end-to-end run follows this order:

```bash
# 1. Generate list of CSV records
python generate_txt.py \
  --root_dir /mnt/project2/DST-SERB/AIIMS-data \
  --out_txt csv_list.txt \
  --log_txt missing_log.txt

# 2. Extract patient-wise data
./data_extract.sh csv_list.txt

# 3. Run inference
./inference.sh

# 4. Convert full ECG records into EDF format
./convert_full_record_into_edf.sh

# 5. Convert inference annotations into EDF annotation format
./convert_into_edf_annotations.sh

# 6. Generate final statistics
python results_stats.py
```

## Visualization in EDF Browser

The repository also supports visualization of ECG records and model-generated annotations in **EDF Browser**.

To visualize results:

1. Convert the full ECG record into EDF format using `convert_full_record_into_edf.py`.
2. Convert inference annotations using `convert_into_edf_annotations.py`.
3. Open the EDF file in EDF Browser.
4. Load the corresponding EDF annotation file.

---

## Notes

* Input ECG records must be in CSV format.
* WFDB records can be converted into CSV using [wfdb_to_csv](https://github.com/barthwalsaurabh0/wfdb_to_csv).
* Update input directories, output directories, and model paths inside the shell scripts before running them.
* `inference.py` is designed for single-patient inference.
* The EDF conversion scripts are mainly used for visualization and manual inspection in EDF Browser.
